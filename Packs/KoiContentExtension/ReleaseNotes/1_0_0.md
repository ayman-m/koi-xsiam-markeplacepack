
#### Parsing Rules

##### KOI Content Extension Parsing Rule

- **New: KOI Content Extension Parsing Rule**, bound to `vendor="koi", product="koi"` and targeting the **koi_koi_raw** dataset with `no_hit=keep`.
- Promotes the item, device, MCP-server and finding attributes that the Marketplace KOI pack delivers only as JSON strings inside *resources*, *observables* and *finding_info* — including *item_id*, *item_type*, *item_marketplace*, *item_risk_level*, *item_package_name*, *item_criticality*, *device_id*, *device_os_obs*, *device_serial*, *device_last_seen*, *device_last_user*, *device_status*, *alert_hostname*, *finding_uid*, *finding_title*, *koi_event_id* and the *mcp_\** family.
- Extraction is coalesce-by-*.type* rather than by array position, because `resources[0].type` is *item* on most alerts and *mcp* on the rest.
- Sets *_time* on Audit rows from *created_at*, guarded by `source_log_type = "Audit"` so alert rows keep the timestamp the integration already supplies.
- Promotes **two marketplace columns in the Koi API vocabulary** — *item_marketplace_api* (Alerts) and *marketplace_api* (Audit) — alongside the untouched raw event values. Koi emits short display forms (*chrome*, *vsc*, *software_windows*) that the Koi API rejects with **HTTP 400**; only *npm* and *pypi* are spelled the same in both vocabularies. The *_api* columns are what a playbook must pass to the *marketplace* argument of `koi-inventory-item-get`, `koi-inventory-item-endpoints-list` and `koi-blocklist-items-add`.
- *built_in*, *side_loaded* and *ollama* map to **null** in the *_api* columns: the first two are *installation_method* values leaking into the *marketplace* field and the third is absent from the API list, so none of them is a valid command argument.
- Note: parsing rules apply at ingest only. Rows already present in **koi_koi_raw** are not reprocessed, so the promoted columns are null for historical data until new events arrive.

#### Modeling Rules

##### KOI Content Extension Modeling Rule

- **New: KOI Content Extension Modeling Rule**, mapping the **koi_koi_raw** dataset to the Cortex Data Model with a separate block for *Alerts* and *Audit* rows.
- The Alerts block depends on the columns promoted by the KOI Content Extension Parsing Rule. The two rules must be deployed together; deployed alone the modeling rule maps nulls.
- *xdm.target.resource.sub_type* now carries the **API** marketplace vocabulary on both blocks (*item_marketplace_api* / *marketplace_api*) instead of the raw short event form, so one XDM field holds one vocabulary and its values are directly usable as command arguments.
- *xdm.observer.name* and *xdm.observer.unique_identifier* gain a **_collector_name fallback**. Koi has stopped emitting *koi_tenant_name* and *koi_customer_id* — populated on 20,324 of 20,522 historical rows but on **zero** rows ingested in the last hours (re-measured 21 July 2026: 19,984 of 20,170 audit rows and 314 of 430 alert rows over 90 days, 0 of 302 rows in the last 24 hours). The original mapping is kept for the historical data; *_collector_name* is 100% populated and preserves the same tenant partition (*plts* ↔ *KOI_PLTS*, *paet* ↔ *KOI_PAET*). Re-check whether the Koi-side change is transient before removing the fallback.

#### Dashboards

##### KOI Content Extension - Alerts Dashboard

- **New: KOI Content Extension - Alerts Dashboard** — 52 widgets over a 30-day relative window, covering alert volume, item and MCP-server risk, device posture and marketplace breakdown. All widgets query **koi_koi_raw** directly and do not require the modeling rule.
- Every marketplace facet is normalised to the Koi API vocabulary and renamed *marketplace_api* / *mcp_registry_api*, so a label an operator reads on a chart is a value they can paste into a `koi-*` command. Values with no API equivalent are labelled rather than silently mixed in: *built_in* / *side_loaded* render as *not-a-marketplace: <value>*, other unknowns as *unmapped: <value>*.
- **Koi MCP Servers by Registry** no longer hides MCP alerts whose registry is blank. It previously filtered them out, showing 11 of 140 MCP alerts; the blanks are now an explicit *(none)* slice, which is itself the security signal (most MCP servers come from no registry).

#### Playbooks

##### KOI Ext - Alert Triage

- **New: KOI Ext - Alert Triage** — end-to-end triage of a KOI alert. Scores four signals, reaches a verdict, and either auto-closes the alert or hands off to response.

##### KOI Ext - Extract Alert Context

- **New: KOI Ext - Extract Alert Context** — sub-playbook that flattens the alert's *finding_info*, *observables* and *resources* into the KoiContext object used by the rest of the triage chain.

##### KOI Ext - Investigate Item

- **New: KOI Ext - Investigate Item** — sub-playbook that investigates one inventory item: the organization's inventory record, the endpoints carrying it, and its allowlist and blocklist standing.

##### KOI Ext - Investigate Device

- **New: KOI Ext - Investigate Device** — device posture check expressed as an inventory query filtered by *device_id*.

##### KOI Ext - Enrich Item

- **New: KOI Ext - Enrich Item** — sub-playbook providing lightweight, reusable enrichment for a single item.

##### KOI Ext - Block and Remediate

- **New: KOI Ext - Block and Remediate** — response playbook that re-checks a flagged item, recovers its *marketplace*, and adds it to the organization blocklist.

##### KOI Ext - MCP Server Audit

- **New: KOI Ext - MCP Server Audit** — standalone audit of the MCP servers present in the KOI inventory.

##### KOI Ext - Unified Script Runner

- **New: KOI Ext - Unified Script Runner** — reads a Script Runner configuration list and dispatches each entry. Additional content: it calls no KOI command, only the Cortex-native ***core-get-scripts***, ***core-get-endpoints*** and ***core-script-run***.

##### KOI Ext - Unified Process Config Entry

- **New: KOI Ext - Unified Process Config Entry** — sub-playbook that validates a single Script Runner configuration entry, executes it, and mails the result through any installed mail-sender integration.

##### KOI Ext - Unified Execute Endpoint Script

- **New: KOI Ext - Unified Execute Endpoint Script** — sub-playbook that resolves the script and target endpoints, runs the script, and polls for its result.

#### Notes

- This pack extends the Palo Alto Networks **KOI** Marketplace content pack (pack *Koi*, integration id *KOI*, version **1.2.3** or later), which must be installed first. It ships no integration and adds no commands of its own — deliberately, to avoid overwriting the installed KOI integration.
- All playbook content is built strictly against the Marketplace pack's **13** commands. There is no `Koi.Device.*` context prefix; a device is addressable only as a filter on inventory (***koi-inventory-list*** with *device_id*).
- Minimum server version **8.4.0**.
